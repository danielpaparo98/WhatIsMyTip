## Follow-ups resolved

The four R-follow-ups flagged in the previous sweep have all been
closed out on this branch.  Totals **unchanged** from the pre-fix run
(60 total / 39 pass / 20 expected-fail / 0 fail / 1 skip), confirming
the changes are pure contract-tightening and doc-correctness, not
behaviour regressions.

### R1 — `POST /api/tips/generate` auth gap

- **Decision:** Added `require_admin_key` as a per-route dependency
  (the endpoint hits OpenRouter and writes to the DB; it should not
  be public).  Wired the existing `X-API-Key` header, updated the
  docs and the endpoint inventory, and re-routed the curl sweep to
  send the admin key for the validation + rate-limited sections.
- **Code:** `backend/app/api/tips.py:257` now declares
  `dependencies=[require_admin_key]`.
- **Tests:**
  - `backend/tests/unit/test_app_api_tips.py:343` (`TestGenerateTips`)
    now asserts missing/wrong/valid keys instead of the old
    "endpoint is open" contract.
- **Docs:** `docs/api.md:163` and
  `plans/integration-test-endpoint-inventory.md:48,157` updated.
- **Sweep delta:** section-3 and section-4 cases now return the
  *expected* status (422 for validation, 200/422/500 for rate-limited)
  instead of short-circuiting to 401.

### R3 — `testcontainers` unit test for `get_by_operation`

- **Decision:** Added a Podman-native testcontainers-style unit test
  (the project uses Podman, not Docker).  The fixture spawns a
  one-shot `postgres:16-alpine` via `podman run`, waits for
  `pg_isready`, runs the test, and tears down with `podman rm -f`.
  Schema is created from the SQLAlchemy model via
  `Base.metadata.create_all`; the runtime engine uses `NullPool` to
  avoid an asyncpg cross-connection state issue.
- **Tests:** `backend/tests/unit/test_generation_progress_crud.py`
  covers all three contract branches (in-progress wins, completed
  fallback, empty) plus the `pending`-ignored edge case, the season
  filter, and the original commit-`103a345` regression
  (`MultipleResultsFound`).
- **Wired into** `backend/scripts/test.sh` automatically
  (`pytest tests/unit/`).  Per-test wall time after container start:
  0.07–0.84s (verified with `--durations=15`).  Module is marked
  `pytest.mark.postgres` and skipped gracefully when Podman is not on
  `PATH`.

### R4 — Tighten `/api/admin/historic-refresh/progress` contract

- **Decision:** Return the in-flight row if any, else the
  most-recently-completed row, else `404 not_found`.  Implemented in
  `GenerationProgressCRUD.get_by_operation` as two single-row
  queries (`status == 'in_progress'`, then
  `status IN ('completed', 'failed')`); the route raises
  `http_error(404, 'not_found', ...)` when both come back empty.
- **Code:** `backend/packages/shared/crud/generation_progress.py:90`
  and `backend/app/api/admin.py:281`.
- **Tests:**
  - `backend/tests/unit/test_app_api_admin.py::TestAdminHistoricRefreshProgress`
    asserts the HTTP contract for all four branches (in_progress /
    completed / failed / not_found).
  - The Postgres-backed unit test in R3 also pins the
    in-flight-first behaviour against real data.
